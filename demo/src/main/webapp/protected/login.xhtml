This quickstart demonstrates how we can costumize PicketLink IDM's to provide a custom authorization checks within an application.

The identity management configuration is based on PicketLink's JPAIdentityStore customized, which uses a database to store applications, users, departments, roles, resource, and some relationships between them.

John has a manager role in New York, Miami and Orlando Departments.

Mary has a sales role in Miami's Department.

Jane has a sales role in Orlando's Department.

A manager role can access acme's resources.

A sales role can access umbrella's and wayne's resources.

Tip: you can login with any of the following username/password combinations:

john/demo
mary/demo
jane/demo
